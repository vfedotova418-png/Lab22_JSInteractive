## Описание

Проект представляет собой интерактивное веб-приложение "Калькулятор", разработанное на JavaScript. Калькулятор имеет веб-интерфейс с кнопками для ввода цифр и операций, дисплеем для отображения выражения и результата. Реализована валидация ввода для предотвращения некорректных математических выражений, а также обработка ошибок вычислений.

## Функциональность

#### Поддерживаемые операции:

- Сложение (+)

- Вычитание (-)

- Умножение (\*)

- Деление (/)

- Десятичные дроби

#### Ограничения:

- Запрещены два оператора подряд.

- Оператор не может находиться в конце выражения.

- Точка не может повторяться в одном числе.

- Первым символом выражения не может быть оператор, кроме минуса.

#### Как обрабатываются ошибки?

- При возникновении ошибки вычисления на дисплее отображается сообщение "Ошибка".

- Калькулятор переводится в состояние hasError = true.

- При нажатии любой кнопки, кроме "C", калькулятор автоматически очищается.

- Кнопка "C" сбрасывает состояние ошибки и очищает дисплей.

#### Описание класса Calculator

Класс реализует всю логику работы калькулятора:

- constructor: инициализирует дисплей, выражение и состояние ошибки

- init(): инициализирует калькулятор, связывая кнопки с логикой .

- handleButtonClick(value): обрабатывает нажатие кнопки

- addToExpression(value): добавляет символ в выражение с валидацией.

- calculate(): вычисляет выражение, обрабатывает ошибки, обновляет состояние калькулятора

- safeEvaluate(expr): выполняет вычисление с предварительной проверкой валидности выражения.

- isValidExpression(expr): проверяет корректность выражения.

- clear(): сбрасывает состояние.

- updateDisplay(): обновляет дисплей.

- showError(message): отображает ошибку.

- isOperator(char): проверяет, является ли символ оператором.

- isLastCharOperator(): проверяет, заканчивается ли выражение оператором.

- hasDuplicateDot(): проверяет наличие двух точек в числе.

#### Схема взаимодействия комплонентов

1. Пользователь нажимает кнопку в index.html

2. main.js ловит событие

3. Вызывается метод handleButtonClick() из calculator.js

4. calculator.js обрабатывает логику (валидация, вычисления)

5. Обновляется display.value через updateDisplay()

6. index.html показывает новый результат на экране

## Логика работы

#### Последовательность действий при нажатии кнопок

1. Пользователь нажимает кнопку

2. Обработчик события получает текст кнопки

3. Проверка состояния ошибки: если hasError = true и нажата не "C", вызывается clear()

4. Анализ кнопки:
   - Если "C" → clear()
   - Если "=" → calculate()
   - Иначе → addToExpression(value)

5. Обновление дисплея через updateDisplay()

#### Как работает валидация ввода?

Валидация реализована в методе addToExpression():

- Запрет двух операторов подряд

- Запрет оператора в начале выражения (кроме минуса)

- Запрет повторной точки в числе

#### Как происходит вычисление?

1. Проверка: if (!this.expression || this.hasError) return;

2. Безопасное вычисление: safeEvaluate(this.expression)
   - Сначала проверяет выражение через isValidExpression()
   - Затем вызывает eval() для получения результата

3. Проверка результата: !isFinite(result) (деление на ноль, бесконечность)

4. Обновление: результат преобразуется в строку и сохраняется в expression

5. Ошибка: при исключении вызывается showError("Ошибка")
